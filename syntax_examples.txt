## Fill triangle (flat color)
## Backface culling: signed_area = (x1-x0)*(y2-y0) - (x2-x0)*(y1-y0); the
## triangle is drawn only if signed_area < 0 (culled/skipped if >= 0).
## Since screen y increases DOWNWARD, this means vertices (x0,y0),(x1,y1),
## (x2,y2) must be ordered COUNTER-CLOCKWISE as seen on screen -- NOT
## clockwise. Example (apex on top, base at bottom, kept/visible):
##   fill_triangle(50,0, 0,100, 100,100, r,g,b);   // top, bottom-left, bottom-right
## Swapping the last two vertices flips the winding and gets it culled.
fill_triangle(x0, y0, x1, y1, x2, y2, r, g, b);

## Key down / key up event handlers. The runtime TRANSLATES the raw virtual
## key to its character (honoring shift/caps/layout) before calling the
## handler, so compare against character or named-key string literals -- not
## raw numbers. A one-char literal is that character (case-sensitive);
## non-character keys have names: "esc" "enter" "backspace" "tab" "space"
## "left" "right" "up" "down" "home" "end" "pageup" "pagedown" "insert"
## "delete" "shift" "ctrl" "alt" "f1".."f12".
on key_down(int key) {
    if (key == "esc") {
        window.close();
    }
    if (key == "a") { }        // lower-case a
    if (key == "A") { }        // Shift+A
    if (key == "left") { }     // arrow key
}
on key_up(int key) { }
